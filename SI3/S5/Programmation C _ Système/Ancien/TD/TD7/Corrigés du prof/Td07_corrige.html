Gestion d’une pile

Nous allons donner ici trois versions de la pile:

  • une version où les fonctions de gestion de la pile renvoient la nouvelle valeur de la pile;
  • une version où l’on passe la pile en paramètre par référence (mais cela nous conduira à avoir des pointeurs sur des pointeurs)
  • une version qui déclare un nouveau type pour éviter d’avoir des doubles pointeurs (dans l’écriture, parce qu’en fait ils existeront bien).

Version 1

Dans cette version, les fonctions push_item et pop_item renvoient la pile après l’ajout ou la suppression d’un élément dans la pile.

Ici nous avons donc pour push_item:

Element *push_item(Element *stack, int n) {
  struct element *new = malloc(sizeof(Element)); // create a new element

  if (!new) {
    fprintf(stderr, "allocation failed\n");
    exit(1);
  }
  new->value  = n;
  new->next   = stack;
  return new;           // return the new stack
}

et pour pop_item nous avons:

Element *pop_item(Element *stack) {
  if (stack) {
    Element *old = stack;

    stack = stack->next;
    free(old);
    return stack;
  }
  fprintf(stderr, "pop on an empty stack!!!\n");
  exit(1);
}

Le problème ici est que nous avons dépilé la «boîte» qui se trouve en sommet de pile, mais que nous n’avons pas de fonction pour trouver l’entier qui se trouvait à l’intérieur de cette «boîte». Il nous faut donc une fonction qui permette de trouver la valeur qui se trouve en sommet de pile:

int top_value(Element *stack) {
  if (stack) {
    return stack->value;
  }
  fprintf(stderr, "top_value on an empty stack!!!\n");
  exit(1);
}

Un exemple d’utilisation de ces fonctions est donné ci-dessous:

  Element *P = NULL;                 // P est une pile vide

  P = push_item(P, 10);              // on ajoute P dans la nouvelle pile
  printf("top: %d\n", top_value(P)); // impression du sommet
  P = pop_item(P);                   // on dépile

Version 2

Dans cette version on veut modifier la pile qui est passée en paramètre. Il faudra donc passer l’adresse de la pile lorsqu’on appelle push_item ou pop_item. Cela veut dire que le type du paramètre correspondant à la pile devra être un pointeur sur la pile (puisqu’elle sera modifiée par l’ajout ou la suppression d’un élément). Comme la pile est représentée par un Element * nous aurons donc un paramètre qui sera de type Element ** (ce qu’il faut lire comme un pointeur sur un Element *).

Du coup, la fonction push_item devient:

void push_item(Element **stack, int n) {
  struct element *new = malloc(sizeof(Element)); // create a new element

  if (!new) {
    fprintf(stderr, "allocation failed\n");
    exit(1);
  }
  new->value  = n;
  new->next   = *stack;
  *stack      = new;            // set the given stack to its new value
}

Pour pop_item on va pouvoir maintenant faire d’une pierre deux coups: la fonction renverra la valeur qui est au sommet tout en ôtant le denier élément empilé. Cette fonction devient:

int pop_item(Element **stack) {
  Element *old = *stack;

  if (old) {
    int old_value = old->value;

    *stack = (*stack)->next;
    free(old);
    return old_value;
  }
  fprintf(stderr, "pop on an empty stack!!!\n");
  exit(1);
}

Noter que la fonction top_value n’est plus vraiment nécessaire avec cette version, puisque la fonction renvoie la valeur qui était en sommet de pile (et la pile est bien modifiée puisqu’on a passé un pointeur en paramètre). Son écriture n’ par contre pas besoin d’être modifiée.

Un exemple d’utilisation de ces fonctions est donné ci-dessous:

  Element *P = NULL;                 // P est une pile vide

  push_item(&P, 10);                 // on ajoute P dans la nouvelle pile
  printf("top: %d\n", pop_item(&P)); // impression du sommet + dépilement

Version 3

Dans cette version, on va définir le type Stack de la façon suivante:

typedef Element *Stack;

Cela permet de donner un nom à la notion «pointeur sur un Element», et cela simplifie (un peu) l’écriture.

Ainsi, les prototypes des fonctions push_item, pop_item et print_stack qui étaient:

void push_item(Element **stack, int n);
int pop_item(Element **stack);
void print_stack(Element *stack);

deviennent:

void push_item(Stack *stack, int n);
int pop_item(Stack *stack);
void print_stack(Stack stack);

Le reste ne change pas.

Listes chaînées

Pour gérer des liste d’entiers, on utilise le même type Element que dans la question précédente et on définit List de la façon suivante:

typedef Element *List;

Commençons par détailler les fonctions simples: find_element et print_element.

Fonction find_element

Cette fonction ressemble fortement aux fonctions de recherche déjà vues précédemment. Une écriture possible pourrait donc être:

int find_element(List lst, int v) {
  for (Element *p = lst; p; p = p->next) {
    if (p->value == v) return 1;
  }
  return 0;
}

Fonction print_list

Là encore, nous avons affaire à une fonction simple à écrire:

void print_list(List lst) {
  printf("{");
  while (lst) {
    printf("%d", lst->value);
    lst = lst->next;
    if (lst) printf(", ");
  }
  printf("}\n");
}

Noter qu’ici on utilise une boucle while plutôt qu’une boucle for car on passe à l’élément suivant avant la fin de la boucle pour savoir si il faut (ou non) afficher une virgule (pas d’affichage si on est sur le dernier élément).

Fonction prepend_element

En fait la fonction prepend_element est identique à la fonction push de la question précédente.

List prepend_element(List lst, int v) {
  // Allouer un nouvel élément et l'initialiser
  Element *tmp = allocate_element();
  tmp->value = v;
  tmp->next  = lst;

  // Renvoyer la nouvelle liste
  return tmp;
}

Fonction append_element

Pour cette fonction on a besoin de trouver le dernier élément de la liste (celui dont le champ next est égal à NULL). Une première écriture à laquelle on pourrait penser pourrait être:

for (p=lst; p->next!=NULL; p=p->next) {
  // rien
}

Cette boucle marche très bien … à condition que l’on soit sûr que lst soit différent de NULL (sinon lst->... provoque une erreur fatale).

La version suivante de append_element s’assure bien que lst n’est pas NULL avant d’utiliser une telle boucle.

List append_element(List lst, int v) {
  // Allouer un nouvel élément et l'initialiser
  Element *tmp = allocate_element();
  tmp->value = v;
  tmp->next  = NULL;

  if (lst == NULL) // Le nouvel elément est la nouvelle liste
    return tmp;

  // lst n'était pas vide. Chercher son dernier élément
  Element *p;
  for (p=lst; p->next; p = p->next) {
  }

  // p désigne le dernier élément; lui faire pointer le nouvel élement
  p->next = tmp;

  // Renvoyer le début de la liste (l'ajout s'est fait plus loin)
  return lst;
}

Fonction insert_element

La fonction insert_element ressemble à une fonction de recherche. En effet, nous devons chercher le premier élément de la liste qui est supérieur à la valeur à insérer. Il nous faudra aussi un pointeur sur l’élément qui précède cette «butée» afin d’insérer le nouvel élément entre les deux (faîtes un dessin pour bien voir ce qui se passe). Nous avons donc:

  Element *prev, *cur;

  // Chercher une valeur > à v dans lst
  for (prev = NULL, cur = lst; cur; prev=cur, cur = cur->next) {
    if (cur->value > v) break;
  }

A la fin de cette boucle, le pointeur sur cette valeur sera dénoté par la variable cur. Noter que cur peut être égal à NULL (on est dans ce cas si aucune valeur de la liste n’est plus grande que v et que, par conséquent l’insertion se fait en fin de liste).

Quand à prev, il dénote l’élément précédent. Si cet élément existe, il faut casser le lien qui relie prev à cur (c’est-à-dire prev->next) et il faut que le suivant du nouvel élément pointe sur cur.

Le code complet de la fonction est donc:

List insert_element(List lst, int v) {
  Element *prev, *cur;

  // Chercher une valeur > à v dans lst
  for (prev = NULL, cur = lst; cur; prev=cur, cur = cur->next) {
    if (cur->value > v) break;
  }

  // Allouer un nouvel élément et l'initialiser
  Element *tmp = allocate_element();
  tmp->value = v;
  tmp->next  = cur;

  if (prev == NULL) // pas de précédent ⇒ le nouvel elément est la nouvelle liste
    return tmp;

  // Chaînage de prec sur tmp
  prev->next = tmp;

  return lst;
}

Fonction delete_element

La fonction de suppression d’un élément commence elle aussi par une recherche.

  • si la recherche est infructueuse, on peut renvoyer la liste originale;
  • si la recherche réussit
    • si l’élément à supprimer et le premier de la liste, il faut renvoyer le pointeur sur le second élément
    • si l’élément à supprimer est «au milieu» de la liste, il faudra faire pointer prev->next sur cur->next pour «sauter» l’élément pointé par cur.

Bien sûr, la fonction ne doit pas oublier de libérer la mémoire occupée par l’élement supprimé.

Nous avons donc:

List delete_element(List lst, int v) {
  Element *prev, *cur, *tmp;

  // Chercher la valeur v dans lst
  for (prev = NULL, cur = lst; cur; prev=cur, cur = cur->next) {
    if (cur->value == v) break;
  }

  if (cur == NULL)              // Pas trouvé
    return lst;
  else {                        // Trouvé
    tmp = cur->next;
    free(cur);

    if (prev == NULL)           //   ... Suppression du premier
      return tmp;
    else {                      //   ... Suppression "au milieu"
      prev->next = tmp;         //       et chaînage
      return lst;
    }
  }
}

Fonction delete_elements

Pour supprimer tous les éléments d’une liste on peut se contenter d’appeler la fonction delete_element tant que l’on sait que l’élément recherché est bien dans la liste.

On obtient une fonction du type:

List delete_elements_v1(List lst, int v) {      //  *** Version 1
    while (find_element(lst, v))
      lst = delete_element(lst, v);
    return lst;
}

Cette version à l’avantage d’être simple, mais elle n’est pas très efficace. En effet à chaque suppression, on cherche (depuis le début de la liste) si v est présent dans la liste. Si c’est le cas, on appelle delete_element (qui repart lui aussi du début de liste).

Une autre façon d’écrire cette fonction consiste à repartir de la fonction qui efface un élément et à l’adapter. Ici, on efface un élément dès qu’on le rencontre. On aura qu’un seul parcours de la liste et on a donc une version plus efficace que la fonction précédente.

La version que l’on obtient est donc:

List delete_elements(List lst, int v) {         //  *** Version 2
  Element *prev, *cur, *tmp;

  prev = NULL;  cur  = lst;
  while (cur) {
    if (cur->value == v) {       //  Trouvé ⇒ supression
      tmp = cur->next;
      free(cur);

      if (prev == NULL)           //   ... Suppression du premier
        lst =  tmp;
      else                        //   ... Suppression "au milieu"
        prev->next = tmp;         //       et chaînage

      // Attention, ne pas faire cur = cur->suivant car le nouveau cur peut
      // lui aussi contenir v (si on a plusieurs v consécutifs)
      cur = tmp;                 // et prev ne bouge pas
    } else {
      prev = cur;
      cur  = cur->next;
    }
  }
  return lst;
}
